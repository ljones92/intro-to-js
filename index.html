Intro to JavaScript

What is JavaScript?

  • Created by Brendan Eich in 1995 as "LiveScript", which was later renamed to "JavaScript"
  • Standardized by the ECMAScript specifications. This introduction covers ES2015(also known as ES6)
  • A client-side processing language. A browser reads the code and runs it directly
  • Combined with HTML & CSS, JavaScript allows you to build dynamic webpages

What can JavaScript do?

Script Tags

You can mix JavaScript and HTML. The script tag tells your browser that what's inside is code, not content 

<script>CODE GOES HERE</script>

JavaScript code can also be moved out to its own file and loaded into HTML with the script tag 

<script src="path/to/file.js"></script>

HTML Selectors

This piece of HTML is simply creating an empty container on the page which has a unique identifier in the form of its ID

An ID is an example of an HTML selector of which there are 3 main types. These selectors allow JS and CSS to interact with the HTML.

ID

IDs should be unique across the entire HTML document and an element can only have a single ID. They are identified with a '#' in CSS and JS

Class

Classes are similar to IDs in their use but they do not need to be unique, multiple elements can possess the same class and a single element can have multiple classes assigned to it. They are identified with a '.' in CSS and JS

Tag

The final main way to identify an element is simply using its tag name directly

Variables

const appDiv = document.getElementById('app');

This line of code is creating a variable named appDiv and initialiasing it with the value returned by document.getElementById('app')

A variable is a container that stores a value and can be reused

There are a few different ways to declare a variable

let

let test = "Hello";

let declares a block scope variable that can be reassigned 


              let test = "Hello";
              console.log(test);
              // expected output: Hello

              test = "Hello Again!"
              console.log(test);
              // expected output: Hello Again!

const

const test = "Hello";

const declares a block scope variable that can not be reassigned 


              const test = "Hello";    
              test = "Hello Again!"
              // TypeError: invalid assignment to const 'test'

var

var test = "Hello";

var declares a function/global scope variable

var was the original way to declare a variable before ES2015 introduced let and const which are the better things to use now

Functions

const appDiv = document.getElementById('app');

document is the entrypoint for JS into the DOM (Document Object Model), it provides a collection of functions to query the HTML. There's a function for each of the selectors we mentioned earlier: 

document.getElementById(id);
document.getElementsByClassName(className);
document.getElementsByTagName(tagName);

In this case we're using getElementById and passing in 'app' which is the value of the id in our HTML.

What the Func is a Function?

A function is a reusable segment of code, they are composed of a sequence of statements called the function body. Values can be passed to a function, and the function will return a value.

Anatomy of a function

                
                  function multiply(x, y) {
                    return x * y;
                  }

                  const value = multiply(2, 7);
                  console.log(value);
                  // expected output: 14

A Quick Note on Data Types

  • string string of characters 
    let userName = "Laurence Jones";
  • number integer or floating point 
    let myAge = 26;
    let yearsAsDeveloper = 3.8;
  • boolean true or false 
    let dogsAreAwesome = true;

More Data Types

  • undefined value that hasn't been defined 
    let favouriteThings;
  • null an explicitly empty value 
    let goodPickupLines = null;
  • symbol a unique identifier 
    let id = Symbol();

Why the Backticks?

Strings in JS are normally wrapped with either ' or ", both working the same

However you can also use a backtick (`) which comes with the nice extra feature of string templating, using ${} allows you to pass variables into a string 


                const myName = "Laurence";
                const myAge = 26;

                console.log(
                  `Hello, my name is ${myName} and I am ${myAge} years old`
                );

                // expected output: Hello, my name is Laurence
                // and I am 26 years old

Handling events in JavaScript

One of the key reasons for using JavaScript is being able to respond to user input to create dynamic web pages, this can be done using `event listeners`

These listeners allow us to react to a wide variety of events including:

  • click
  • scroll
  • keydown

And many more

Writing an event listener

Say we have a button with id 'button', we could create a click event for this button by doing the following: 

              
                const button = document.getElementById('button');

                button.addEventListener('click', function() {
                  console.log('You\'ve clicked the button!');
                });

The if statement

Use if statements to decide which lines of code to execute, based on a condition.


              if (condition) {
                // statements to execute
              }

Comparison Operators

Example Name Result
a == b Equal TRUE if a is equal to b (can be different types).
a === b Identical TRUE if a is equal to b, and the same type.
a != b Not equal TRUE if a is not equal to b (can be different types).
a !== b Not identical TRUE if a is not equal to b, or they are not the same type.
a < b Less than TRUE if a is strictly less than b.
a > b Greater than TRUE if a is strictly greater than b.
a <= b Less than or equal to TRUE if a is less than or equal to b.
a >= b Greater than or equal to TRUE if a is greater than or equal to b.

Watch out!

Don't mix up = and == and ===

The if/else statement

Use else to provide an alternate set of instructions.


              const age = 26;
            
              if (age >= 18) {
                console.log('Have a beer!');
              } else {
                console.log('No beer for you');
              }

The if/else statement

If you have multiple conditions, you can use else if.


              const pet = "dog";
              
              if (pet === "dog") {
                console.log('You have the best type of pet');
              } else if (pet === "cat") {
                console.log('Better than nothing');
              } else {
                console.log('Get a dog');
              }
